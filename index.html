JSMinjector

This is an easy setup for unit-testing Javascript based on the popular Jasmine BDD suite. The idea being that introducing unit-testing to developers that are unaware of it's use should be as painless and easy as possible. Thus this attempt to inject Jasmine into any page using a single Bookmarklet.

Of course that in itsself would still be rather useless, so the developer will also need to provide a location where the 'specs' can be found and a single specs.js file is expected to be there to provide an array of specs to run. The injection should take it from there, adding the additional specs to the page and finally firing up Jasmine.

Some added CSS should help displaying the result on top of your web-page :) So, in summary, what do you need to do to make use of this neat package:

  • Store this JSMinjector Bookmarklet in your browser of choice (tested only with Chrome, but 'should' work across the board...)
  • Create some test 'specs' and place them in a folder on your to-be-tested application, or anywhere else that you can access from your browser.
  • Open the page on which you want to let lose the unit-tests.
  • Click on the bookmarklet and provide the location of your spec files.
  • Check the results
